  - `flutter-apps/dandpak_desktop` (Windows/Linux/macOS)
  - `flutter-apps/dandpak_tablet` (Android/iOS)
  - `flutter-apps/dandpak_phone` (Android/iOS)

- Chuyển asset UI từ `assets/web/assets` sang `assets/brand`.
- **Gỡ trùng lặp triệt để (2026-07-13):** trước đó 3 app là bản sao gần như y hệt
  (105 file dart × 3, chỉ khác 1 chuỗi app-id). Nay TOÀN BỘ code dùng chung (models,
  providers, screens, services, ui, widgets, utils, bootstrap, root app widget) dồn về
  MỘT nơi `flutter-apps/dandpak_core/lib/src/`. Ba app chỉ còn **vỏ mỏng** 2 file:
  `lib/main.dart` + `lib/app_version.dart`.
- **AppFlavor** (`dandpak_core/lib/src/app_flavor.dart`) là điểm phân hoá DUY NHẤT giữa
  3 app: `appId`, `versionName`/`buildNumber` (từ app_version.dart), `enabledModuleKeys`
  (bộ module theo thiết bị — desktop = tất cả, tablet/phone = bộ riêng → "khác số lượng
  module"), `layout` (`station`/`tablet`/`handset` — cùng ngôn ngữ thiết kế, khác bố cục).
  Mỗi `main.dart` gọi `runDandpakApp(flavor: AppFlavor(...))`.
- Plugin (media_kit, mobile_scanner, desktop_multi_window, video_player_win…) khai trong
  `dandpak_core/pubspec.yaml`, kéo vào 3 app qua phụ thuộc bắc cầu; app chỉ giữ `assets`
  + `fonts` + thư mục nền tảng. Unit test dùng chung ở `dandpak_core/test/`.
- Hộp đen/nhật ký (black_box, client_log, system_log, app_updater) đọc `AppFlavor.current`
  thay cho hằng số app_version cứng → biết lỗi thuộc app nào mà không nhân bản mã.

## Ghi Chú Build

- Lỗi `ninja: ...Microsoft.Windows.ImplementationLibrary.targets missing` khi build
  `dandpak_desktop` là do bước nuget restore của plugin `video_player_win` không tạo được
  file `.targets` dưới flutter + VS 2026 (v18). Gói WIL `1.0.220914.1` ĐÃ có trong nuget
  cache nên restore chạy được offline — cần chạy đúng môi trường build của máy.
- Cách build ổn định (đã dùng cho các bản trước): chạy `flutter-apps/dandpak_desktop/
  build_release.bat` (đã có sẵn `vcvars64` + `flutter build windows --release`) trong
  shell tương tác. Nếu vẫn kẹt nuget: `flutter clean` rồi build lại, và đảm bảo `nuget.exe`
  + thư mục VS Installer (`vswhere.exe`) nằm trên PATH.
- Bản Android (`dandpak_tablet`/`dandpak_phone`) không dùng nuget nên không vướng lỗi này;
  đồ thị plugin Android đã xác nhận đúng qua GeneratedPluginRegistrant.
